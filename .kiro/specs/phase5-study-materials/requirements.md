# 要件定義: Phase 5 - 学習資料機能

## 概要

**目的**: プログラミング初学者が「まず読んで学び、次に解く」という自然な学習フローを体験できるよう、演習問題（Lesson）に先行する学習資料（Material）機能を追加する。

**対象ユーザー**:
- 学習者: 資料を読んでから演習に取り組む
- 管理者: Markdownで資料を作成・編集する

---

## ユーザーストーリー

### 学習者向け

**US-1: カテゴリページで学習資料を確認する**
> As a 学習者 / I want to カテゴリページで学習資料と演習問題の一覧を確認したい / So that 何を学べるかが一目でわかる。

受け入れ条件:
- [ ] カテゴリページに「学習資料」セクションが表示される
- [ ] 学習資料は順序付きで一覧表示される（order順）
- [ ] 演習問題セクションは学習資料セクションの下に表示される
- [ ] 資料タイトルをクリックすると資料詳細ページへ遷移する

**US-2: 学習資料を読む**
> As a 学習者 / I want to 学習資料をブラウザ上で読みたい / So that 環境構築なしに内容を学べる。

受け入れ条件:
- [ ] Markdownがリッチにレンダリングされる（見出し・コードブロック・表・リストなど）
- [ ] コードブロックにシンタックスハイライトが適用される
- [ ] 資料下部に「演習問題を解く」ボタンが表示される（同カテゴリのLesson一覧へ）
- [ ] 「前の資料」「次の資料」のナビゲーションがある（order順）
- [ ] ページタイトルに資料名が表示される

### 管理者向け

**US-3: 学習資料を作成する**
> As a 管理者 / I want to Markdownで学習資料を作成したい / So that AIが生成したコンテンツを簡単に登録できる。

受け入れ条件:
- [ ] 管理画面から学習資料の新規作成ができる
- [ ] タイトル・カテゴリ・表示順・Markdownコンテンツを入力できる
- [ ] 保存後、管理一覧ページへリダイレクトされる

**US-4: 学習資料を編集・削除する**
> As a 管理者 / I want to 既存の学習資料を編集・削除したい / So that コンテンツを最新状態に保てる。

受け入れ条件:
- [ ] 管理一覧から編集ページへ遷移できる
- [ ] 既存の内容が編集フォームに初期値として入っている
- [ ] 削除は確認ダイアログを経て実行される

## シーダーデータ要件

**JS-1: JavaScript基礎コース（10本以上）**
- 変数とデータ型（`var`/`let`/`const`、型の種類）
- 演算子（算術・比較・論理）
- 条件分岐（`if/else`、`switch`）
- ループ（`for`、`while`、`for...of`）
- 関数（関数宣言・アロー関数）
- 配列（作成・操作・メソッド）
- オブジェクト（プロパティ・メソッド）
- スコープとクロージャ
- 非同期処理入門（`setTimeout`、コールバック）
- DOM操作入門

**JS-2: PHP基礎コース（10本以上）**
- 変数とデータ型（`$変数`、型の種類）
- `echo` / `print` による出力
- 演算子（算術・比較・論理・文字列）
- 条件分岐（`if/else`、`match`）
- ループ（`for`、`while`、`foreach`）
- 関数（定義・引数・戻り値）
- 配列（インデックス配列・連想配列）
- 文字列操作（主要な組み込み関数）
- 型変換とキャスト
- クラスとオブジェクト入門

各コースに対応する演習問題（Lesson）も合わせて作成すること。

---

## 非機能要件

- **レンダリング**: Markdownは `react-markdown` + `@tailwindcss/typography`（proseクラス）+ `rehype-highlight` でレンダリングする
- **セキュリティ**: Markdownのレンダリング時にHTMLサニタイズを適用する（`rehype-sanitize` または `allowedElements` 制限）
- **パフォーマンス**: 資料詳細ページはSSRまたはSSGで提供する（Next.jsの機能を活用）
- **アクセス制御**: 学習資料の閲覧はログイン済みユーザーのみ。作成・編集・削除は管理者のみ。
