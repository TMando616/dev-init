# Requirements: Phase 3 - Dashboard, Progress & Categories

## 1. 概要
本フェーズでは、ユーザーの学習意欲を高め、学習状況を可視化するための「ダッシュボード」と「進捗管理機能」、およびコンテンツ整理のための「カテゴリ機能」を実装します。

## 2. 機能要件

### 2.1 レッスンの進捗管理
- **手動完了報告**: ユーザーが演習画面で「完了」ボタンを押すことで、そのレッスンの進捗を記録する。
- **進捗ステータス**: 未着手、学習中（コード保存あり）、完了の3状態を管理。
- **UI表示**: レッスン一覧やダッシュボードで、完了済みのレッスンにチェックマーク等のバッジを表示する。

### 2.2 レッスンのカテゴリ管理 (マスタ管理)
- **カテゴリマスタ**: `categories` テーブルを新設し、名前（必須）や説明（任意）を保持する。
- **管理者用CRUD**: 管理画面からカテゴリの作成・一覧表示・編集・削除ができる。
- **レッスンとの紐付け**: レッスンは必ず1つのカテゴリに属する（1対多）。
- **ユーザー画面**: レッスン一覧をカテゴリごとにグループ化して表示する。

### 2.3 ユーザーダッシュボード (トップページの置換)
- **デフォルト画面**: ログイン後のトップページ（`/`）をダッシュボードに置き換える。
- **表示内容**:
    - 全体進捗：円グラフやプログレスバーによる視覚的表示（完了数 / 全レッスン数）。
    - カテゴリ別進捗：各カテゴリごとの進捗状況。
    - 最近の活動：最後に学習したレッスンの継続ボタン。

## 3. 検討・相談事項 (解決済み)
- **完了判定**: 手動ボタン方式を採用。
- **カテゴリ**: マスタテーブルによる厳密な管理を採用。
- **配置**: トップページをダッシュボードに変更。
