# 要件定義: Phase 9 - 学習資料管理画面の改善

## 概要

**目的**: 管理者向け「学習資料」まわりの管理画面を、資料件数が増えても運用しやすい状態に改善する。

**背景・課題**:
- 学習資料エディタのレッスン選択（`/admin/materials/[id]`）が全レッスンをフラットに一覧表示しており、レッスン数が増えると目的のレッスンを探しにくい。
- 学習資料一覧（`/admin/materials`）はレッスン列のみで、どのカテゴリに属する資料かが一覧上わからない。
- 学習資料一覧に検索・絞り込みがなく、全件表示のみ。件数が増えると目視で探すしかない。
- 学習資料の表示順（`order`）はフォーム上の数値入力のみで、同一レッスン内での重複や欠番を防ぐ仕組みがない。
- 初期シードデータが JavaScript・PHP の2言語分しかなく、実行エンジンが対応済みの Python・Java にはレッスン・学習資料が1件もない。

**今回の方針決定（確定事項）**:
- Phase 8 で確立した「カテゴリ ↔ レッスン（多対多）」「レッスン → 学習資料（1対多）」の構造は変更しない。今回はこの構造の上での**管理画面UXの改善**に限定する。
- `GET /lessons` が既に `categories:id,name` を返しているため、US-1（カテゴリ絞り込み）はフロントエンドのみで実現できる。バックエンド変更は不要。
- US-2（カテゴリ列表示）は `GET /materials` のレスポンスに `lesson.categories` を含める必要があり、バックエンド変更を伴う。
- US-5（シーダー追加）は **Python・Java のみ**を対象とする。React入門・Webデザインカテゴリは既存シーダー（`DatabaseSeeder`）にカテゴリのみ存在しレッスン0件のままだが、コード実行エンジン（`CodeExecutionService::$languages`）が `php/python/javascript/ruby/java` の5言語にしか対応しておらず、HTML/CSS・JSXを実行できないため今回はスコープ外とする（将来、実行エンジン拡張とあわせて別要件で扱う）。

---

## ユーザーストーリー

### US-1: 学習資料エディタでカテゴリからレッスンを絞り込む
> As a 管理者 / I want to 学習資料の作成・編集画面でカテゴリを選んでからレッスンを選びたい / So that レッスン数が多くても目的のレッスンをすぐ見つけられる。

受け入れ条件:
- [ ] 学習資料エディタ（`/admin/materials/[id]`）に「カテゴリで絞り込む」選択肢が追加される
- [ ] カテゴリ未選択（デフォルト）の場合、レッスン選択肢は現状通り全件表示される（既存資料編集時に紐づくレッスンが選択肢から消えない）
- [ ] カテゴリを選択すると、レッスン選択肢がそのカテゴリに属するレッスンのみに絞り込まれる
- [ ] カテゴリを切り替えた結果、現在選択中のレッスンが絞り込み後の選択肢に含まれなくなった場合、レッスン選択は未選択状態に戻る
- [ ] レッスン選択は引き続き必須（未選択での保存は拒否）

### US-2: 学習資料一覧でカテゴリを確認する
> As a 管理者 / I want to 学習資料一覧で各資料がどのカテゴリのレッスンに属するか確認したい / So that レッスン列だけでは分からないカテゴリ単位の分布を把握できる。

受け入れ条件:
- [ ] 学習資料一覧（`/admin/materials`）に「カテゴリ」列が追加される
- [ ] 各行に、紐づくレッスンが持つカテゴリ名がタグ形式で表示される（1レッスンが複数カテゴリに属する場合は複数タグ）
- [ ] レッスン未設定 or カテゴリ未設定の資料は「—」で表示される
- [ ] `GET /api/materials` のレスポンスに `lesson.categories` が含まれるようバックエンドを更新する（`MaterialRepository::all()` の eager load 修正）

### US-3: 学習資料一覧で検索・絞り込みができる
> As a 管理者 / I want to 学習資料一覧をタイトル・レッスン・カテゴリで絞り込みたい / So that 件数が増えても目的の資料をすぐ見つけられる。

受け入れ条件:
- [ ] 学習資料一覧にタイトルの部分一致検索ボックスが追加される
- [ ] レッスンでの絞り込みセレクトが追加される
- [ ] カテゴリでの絞り込みセレクトが追加される（US-2 で取得するカテゴリ情報を利用）
- [ ] 検索・絞り込みは組み合わせ可能（AND条件）
- [ ] 絞り込み結果が0件の場合、その旨のメッセージを表示する
- [ ] 実現方式はクライアントサイドの絞り込みとする（既存 `GET /materials` は全件取得のままでよい。ページネーションはスコープ外）

### US-4: 学習資料の表示順（order）の重複・欠番を防ぐ
> As a 管理者 / I want to 同一レッスン内で表示順が重複した状態を作れないようにしたい / So that 意図しない並び順になることを防げる。

受け入れ条件:
- [ ] 同一 `lesson_id` 内で `order` が重複する内容を保存しようとした場合、保存前に検知できる（実現方式はドラッグ&ドロップ並び替え／自動採番／重複時のインラインエラーのいずれかとし、設計フェーズで確定する）
- [ ] 学習資料エディタで対象レッスンを選択した際、そのレッスン内の既存資料の order 一覧（またはドラッグ&ドロップ用の並び）を参照できる
- [ ] 既存データ（Phase 8 までのシーダー・登録済みデータ）に対して後方互換性を壊さない（マイグレーション不要、または安全なマイグレーションを設計フェーズで検討）

### US-5: Python・Java のレッスン・学習資料・カテゴリをシーダーで追加する
> As a 管理者 / I want to Python と Java のカテゴリ・レッスン・学習資料が初期データとして揃っている状態にしたい / So that `migrate:fresh --seed` 直後から JavaScript・PHP と同じ水準で Python・Java の演習を提供できる。

受け入れ条件:
- [ ] `DatabaseSeeder` に「Python基礎」「Java基礎」カテゴリが追加される（既存の「JavaScript基礎」「PHP基礎」と同じ形式: `name` + `description`）
- [ ] `LessonSeeder` に Python 用レッスンが追加される。既存 `$jsLessons`/`$phpLessons` と同じ構成（`title` / `content` / `model_answer` / `expected_output` / `language: 'python'`）で、Hello World → 変数 → 算術演算 → 条件分岐 → ループ → 関数 → 配列/リスト → 辞書 → クラス → 応用（1トピック）の10件とし、「Python基礎」カテゴリに `sync` する
- [ ] `LessonSeeder` に Java 用レッスンが追加される。同様の構成で `language: 'java'` の10件とし、「Java基礎」カテゴリに `sync` する
- [ ] `MaterialSeeder` に Python 用学習資料が追加される。既存 `$jsMaterials`/`$phpMaterials` と同じ構成（`title` / `order` / `content`(Markdown)）で10件とし、既存パターンに倣い `language('python')` の最初のレッスンに `lesson_id` として紐づける
- [ ] `MaterialSeeder` に Java 用学習資料が追加される。同様の構成で10件とし、`language('java')` の最初のレッスンに紐づける
- [ ] `docker compose exec php php artisan migrate:fresh --seed` が成功する
- [ ] シード後、管理画面のレッスン一覧・学習資料一覧・カテゴリ一覧に Python・Java のデータが表示される（US-1〜4 のカテゴリ絞り込み・列表示・検索でも正しく扱えることを確認）
- [ ] 生徒側の演習ページで Python・Java レッスンが実際に「実行」でき、`expected_output` と一致する（既存の Docker サンドボックス実行が問題なく通ること）
- [ ] 既存の JavaScript・PHP のレッスン・資料・カテゴリの内容や件数に影響を与えない（回帰なし）

## ギャップ分析（既存コードとの乖離）

| 領域 | 現状 | 本要件での変更 |
|------|------|----------------|
| `GET /lessons` | `categories:id,name` を含めて返却済み | 変更なし（US-1 はフロントのみ） |
| `GET /materials` | `lesson:{id,title}` のみ返却 | `lesson.categories` を追加（US-2） |
| `MaterialRepository::all()` | `Material::with('lesson:id,title')` | `lesson.categories:id,name` を追加 eager load |
| 管理UI (`/admin/materials/[id]`) | レッスン選択がフラットな全件セレクト | カテゴリ絞り込みセレクトを追加（US-1） |
| 管理UI (`/admin/materials`) | タイトル・レッスン・順序・操作の4列、検索なし | カテゴリ列追加、検索・絞り込みUI追加（US-2, US-3） |
| `order` の重複制御 | フォームの数値入力のみ、バリデーションなし（`findPrevNext` は id を tie-break にして表示上は破綻しないが、意図した並びにならない） | 重複防止の仕組みを追加（US-4、方式は設計フェーズで決定） |
| `DatabaseSeeder` | カテゴリは JS基礎/PHP基礎/React入門/Webデザインの4件（React入門・Webデザインはレッスン0件） | Python基礎・Java基礎の2カテゴリを追加（US-5） |
| `LessonSeeder` | JavaScript・PHP のみ各10レッスン | Python・Java を各10レッスン追加（US-5） |
| `MaterialSeeder` | JavaScript・PHP のみ各10資料（各言語の最初のレッスンに集約して紐づけ） | Python・Java を同パターンで各10資料追加（US-5） |

---

## スコープ外（今回やらない）

- 学習資料一覧のサーバーサイドページネーション（件数が現状少なく、クライアントサイド絞り込みで十分なため）
- レッスン↔カテゴリ、資料↔レッスンの関係構造そのものの変更（Phase 8 の構造を維持）
- レッスン編集画面から学習資料を直接作成・並び替えする逆方向の導線（今回は学習資料エディタ側の改善に限定）
- React入門・Webデザインカテゴリへのレッスン・資料追加（実行エンジンが対応言語外のため。実行エンジン拡張は別要件で検討）
