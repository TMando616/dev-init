# Requirements: Phase 4 - Multi-Language Code Execution

## 1. 概要
本フェーズでは、ユーザーがブラウザ上で記述したコードを、サーバー上の隔離された環境で実行し、その出力をリアルタイムに表示する機能を実装します。

## 2. 対象言語
以下の 5 言語を初期サポート対象とします。
- **PHP**
- **Python**
- **JavaScript (Node.js)**
- **Java**
- **Ruby**

## 3. 機能要件

### 3.1 コード実行 (Backend)
- **隔離環境での実行**: Docker コンテナを利用し、Web サーバーやデータベースから分離された環境でユーザーコードを実行する。
- **言語別実行エンジン**: 各言語に対応した軽量な Docker イメージまたは単一の多言語イメージを使用する。
- **実行制限**:
    - **タイムアウト**: 実行開始から一定時間（デフォルト 3 秒）経過しても終了しない場合は強制終了する。
    - **メモリ・ネットワーク制限**: 悪意のあるコードによるリソース消費を抑える（可能な範囲で設定）。
- **結果取得**: 標準出力 (stdout) および標準エラー出力 (stderr) をキャプチャし、実行ステータス（成功・失敗・タイムアウト）と共にフロントエンドに返す。

### 3.2 演習画面の拡張 (Frontend)
- **「実行」ボタン**: 演習画面のエディタエリアに「実行 (Run)」ボタンを追加する。
- **コンソールエリア**: エディタの下部または横に、実行結果を表示する黒い背景のコンソール領域を追加する。
- **言語選択 (管理者)**: レッスン作成時に、そのレッスンで使用する言語を指定できるようにする。
- **ステータス表示**: 実行中（Loading）、成功、エラー、タイムアウトなどの状態を視覚的に表示する。

### 3.3 データモデルの拡張
- `lessons` テーブルに `language` カラムを追加し、実行に使用する言語を保持する。

## 4. 非機能要件
- **セキュリティ**: ユーザーコードがホストマシンや他のコンテナにアクセスできないようにする。
- **応答性**: コード実行リクエストに対して、ネットワーク遅延を除き、実行終了後速やかに結果を返す。

## 5. アウトオブスコープ (本フェーズでは実施しない)
- **自動判定**: 期待される出力と一致するかどうかのチェック。
- **対話型入力**: `stdin` を介したユーザーとの対話。
- **ファイル操作**: ユーザーによる複数ファイルの作成や永続化。
- **ネットワークアクセス**: ユーザーコードからの外部ネットワークへのアクセス。
