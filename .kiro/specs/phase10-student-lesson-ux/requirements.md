# 要件定義: Phase 10 - 受講生向けレッスンUX改善

## 概要

**目的**: 受講生（ユーザー）側のレッスン閲覧・学習導線を改善し、進捗の可視化と学習の連続性を高める。

**背景・課題**:
- レッスン一覧（`/lessons/list`）・カテゴリ別レッスン一覧（`/categories/[id]`）は、どのレッスンが完了済みかを一覧上で判別できない。
- レッスン詳細画面（`/lessons/[id]`）はレッスン単体の完結した画面になっており、次のレッスンに進むには一覧画面に戻る必要がある。
- ユーザー向けサイドバー（`Sidebar.tsx`）は「ダッシュボード」「レッスン一覧」の固定2項目のみで、カテゴリへのショートカットがない（管理画面の `AdminSidebar.tsx` も同様にカテゴリは管理画面への単一リンクのみで、動的なカテゴリ一覧ではない）。

**今回の方針決定（確定事項）**:
- 本要件は受講生向け改善のうち、現時点で合意している3件（US-1〜US-3）のみを対象とする。ユーザー側の改善は継続的に追加予定であり、本ドキュメントは全体の一部（第1弾）である。
- Phase 8 で確立した「カテゴリ ↔ レッスン（多対多）」「レッスン → 学習資料（1対多）」の構造は変更しない。

---

## ユーザーストーリー

### US-1: レッスン一覧で完了済みレッスンに完了マークを表示する
> As a 受講生 / I want to レッスン一覧・カテゴリ別レッスン一覧で完了済みのレッスンがひと目でわかるようにしたい / So that 未着手のレッスンを見分けて次に取り組むべきものを判断できる。

受け入れ条件:
- [ ] レッスン一覧（`/lessons/list`）のレッスンカードに、完了済み（`Submission.status = 'completed'`）の場合は完了マーク（アイコン等）が表示される
- [ ] カテゴリ別レッスン一覧（`/categories/[id]`）のレッスンカードにも同様に完了マークが表示される
- [ ] 未着手・保存済み（未完了）のレッスンには完了マークは表示されない
- [ ] レッスン件数分の完了状況をまとめて取得できる必要がある（現状 `GET /lessons` `GET /categories/{id}` はレッスンデータのみを返し、認証ユーザーごとの完了状況を含まない。個別に `GET /submissions/lesson/{id}` をレッスン数分呼ぶのは非効率なため、返却形式は設計フェーズで確定する）

### US-2: レッスン画面から次のレッスンへ遷移する
> As a 受講生 / I want to レッスンを開いている画面から次のレッスンへワンクリックで遷移したい / So that 一覧画面に戻らずに連続して学習を進められる。

受け入れ条件:
- [ ] レッスン詳細画面（`/lessons/[id]`）のヘッダー右上に「次のレッスンへ」ボタンが表示される
- [ ] ボタン押下で次のレッスンの詳細画面（`/lessons/{next_id}`）に遷移する
- [ ] 次のレッスンが存在しない場合（一覧の最後のレッスン）、ボタンは非表示または無効化状態にする
- [ ] 「次」の並び順の定義（同一カテゴリ内での順序か全体順か、`id` 順かなど）を設計フェーズで確定する。現状 `Lesson` モデルには `Material` の `order` に相当するカラムが存在しないため、順序の基準を新たに定義する必要がある
- [ ] ボタンはレッスンが完了済み（`isCompleted`）の場合のみ活性化し、未完了の場合は非活性（disabled）にする

### US-3: ユーザーサイドバーにカテゴリを表示する
> As a 受講生 / I want to サイドバーからカテゴリ別レッスン一覧に直接アクセスしたい / So that レッスン一覧画面を経由せずに目的のカテゴリへ素早く移動できる。

受け入れ条件:
- [ ] サイドバー（`Sidebar.tsx`）に、登録されているカテゴリの一覧（`GET /categories`）が動的に表示される
- [ ] 各カテゴリ項目をクリックすると `/categories/{id}` に遷移する
- [ ] 現在表示中のカテゴリに対応する項目はアクティブ状態でハイライトされる（既存のダッシュボード・レッスン一覧のハイライトと一貫した見た目にする）
- [ ] カテゴリが0件の場合、カテゴリセクションは表示しない（もしくは適切な空状態にする）
- [ ] サイドバーの折りたたみ（`isCollapsed`）状態でも、既存ナビ項目と一貫した挙動・見た目にする

## ギャップ分析（既存コードとの乖離）

| 領域 | 現状 | 本要件での変更 |
|------|------|----------------|
| `GET /lessons`（`LessonRepository::all()`） | 全ユーザー共通のレッスン一覧（`categories:id,name` を含む）を返却。ユーザーごとの完了状況は含まない | 認証ユーザーの完了状況を含める、または別途一括取得する手段を追加（US-1） |
| `GET /categories/{id}`（`CategoryController::show`） | カテゴリに紐づく `lessons`（`categories` 含む）を返却。完了状況は含まない | 同上（US-1） |
| `Lesson` モデル・`lessons` テーブル | `order` 相当のカラムが存在せず、並び順は `id`／挿入順に依存 | 「次のレッスン」を決定するための順序基準を新たに設計する（US-2） |
| レッスン詳細ヘッダー（`frontend/src/app/(student)/lessons/[id]/page.tsx`） | 「模範解答」「保存」「実行」「完了にする」ボタンのみ。次レッスンへの導線なし | 「次のレッスンへ」ボタンを追加（US-2） |
| `Sidebar.tsx` | `navItems` は「ダッシュボード」「レッスン一覧」の固定配列のみ。カテゴリ表示なし | `GET /categories` を取得し、動的にカテゴリ項目を追加（US-3） |
| `frontend/src/app/(student)/lessons/list/page.tsx` / `.../categories/[id]/page.tsx` | `LessonCard` はタイトル・カテゴリタグのみ表示 | 完了マーク表示のためのUI変更（US-1） |

---

## スコープ外（今回やらない）

- US-1〜US-3以外の受講生向けUX改善（別途追加予定、今回は含めない）
- レッスン自体の並び順（`order`）を管理画面から編集する機能（US-2で順序基準が決まった場合でも、編集UIはスコープ外）
- ダッシュボード画面自体のレイアウト変更
