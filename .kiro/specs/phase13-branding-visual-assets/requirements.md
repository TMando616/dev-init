# Phase 13: ブランディング素材（ロゴ・言語トップ画像）の導入

## 概要

正式なロゴ画像（`dev-init-logo.png`）と、対応言語のうちPHP/Python/JavaScript/Javaの4言語分のトップ画像が用意された。これらをアプリ内の主要な画面に反映し、プレースホルダー的な見た目（アイコン+テキストのみのロゴ、言語表示のないレッスン一覧）から脱却する。対象ユーザーは受講生・管理者の両方。

## 素材

| ファイル | サイズ | 用途 |
|---------|--------|------|
| `dev-init-logo.png` | 2020×773 | ロゴ（アイコン+「dev-init」テキスト一体型、白背景） |
| `javascript-logo.png` / `python-logo.png` / `java-logo.png` / `php-logo.png` | 各2752×1536 | 言語トップ画像（アイコン+言語名+キャッチコピー、右下に`dev-init`ロゴ埋め込み済みの横長バナー） |
| `ruby-logo.png` | 3352×1280 | 言語トップ画像（同上のデザイン。他4枚とアスペクト比が異なる点は設計フェーズで正規化を検討） |
| `dev-init-favicon.ico` | 24×24 | ブラウザタブfavicon |

元ファイルは `/mnt/c/Users/user/Downloads/` にある。対応言語（PHP/Python/JavaScript/Ruby/Java）の画像が**5言語すべて揃った**。

## ユーザーストーリー

### US-1: ロゴの正式反映

> As a 受講生・管理者 / I want アプリのヘッダー・サイドバーとログイン・登録画面に正式なdev-initロゴが表示されてほしい / So that プレースホルダー的な見た目から脱却し、ブランドイメージが統一される。

受け入れ条件:
- [ ] `Sidebar.tsx`（学生側）のロゴ部分（現状: アイコンボックス+「DevInit」テキスト）を`dev-init-logo.png`に差し替える
- [ ] `AdminSidebar.tsx`（管理者側）も同様に差し替える
- [ ] 学生ログイン画面（`(student)/login`）・登録画面（`(student)/register`）の見出し（現状: `<h1>DevInit</h1>`テキスト）をロゴ画像に差し替える
- [ ] 管理者ログイン画面（`(admin)/admin/login`）も同様に差し替える
- [ ] ダークモード時（`prefers-color-scheme: dark`）でも視認性が破綻しない（ロゴ画像は白背景前提のため、配置先の背景色との相性を確認し、必要に応じて背景に白いカード/枠を敷くなどの対応を行う）
- [ ] サイドバー折りたたみ時（`isCollapsed`）は、現状のアイコンのみ表示ロジックに準じた縮小表示を維持する
- [ ] `frontend/src/app/favicon.ico`（Next.js App Routerの規約ファイル）を`dev-init-favicon.ico`に差し替える

### US-2: レッスン一覧カードへの言語画像の活用

> As a 受講生 / I want レッスン一覧の各カードに、そのレッスンが何の言語かひと目でわかる画像が表示されてほしい / So that 学習前にどの言語のレッスンか直感的に判断できる。

受け入れ条件:
- [ ] `LessonCard`（`(student)/lessons/list/page.tsx`）の左側アイコン欄（現状: `BookOpen`アイコンの正方形ボックス）を、レッスンの`language`フィールドに応じた言語トップ画像のクロップサムネイルに拡張する
- [ ] `Lesson`インターフェースに`language`フィールドを追加し、API（`/lessons`）のレスポンスから取得する
- [ ] 5言語（PHP/Python/JavaScript/Ruby/Java）以外の未知の`language`値の場合は、現状の`BookOpen`アイコン表示にフォールバックする（表示が破綻しない）

### US-3: 言語別セクションの新設（ダッシュボード・レッスン一覧）

> As a 受講生 / I want ダッシュボードとレッスン一覧に「言語別」の新セクションがあり、言語トップ画像とレッスン数が表示されてほしい / So that 言語ごとの学習ボリュームを俯瞰し、興味のある言語から学習を始められる。

受け入れ条件:
- [ ] カテゴリ名からの言語自動判定は行わない（カテゴリと言語は独立した軸として扱う）
- [ ] `(student)/page.tsx`（ダッシュボード）に、既存の「カテゴリ別進捗」セクションとは別に「言語別」セクションを新設し、`Lesson.language`で集計したレッスン数を言語トップ画像付きカードで表示する
- [ ] `(student)/lessons/list/page.tsx`（レッスン一覧）にも同様の言語別グルーピング導線を追加する（既存のカテゴリ別グルーピング表示と共存させる。具体的な配置・UIは設計フェーズで決定）
- [ ] レッスン数が0件の言語はセクション内に表示しない
- [ ] 管理画面（`(admin)/admin/...`）への同様の言語別セクション追加は本フェーズのスコープ外とする

### US-4: 未知の言語値へのフォールバック

> As a 開発者 / I want 言語→画像のマッピングにない`language`値が来ても表示が破綻しない仕組みがほしい / So that 今後6言語目が追加される際もマッピングに1件追加するだけで対応でき、データ不整合時にも壊れない。

受け入れ条件:
- [ ] 言語→画像のマッピングを1箇所（定数/設定）に集約し、追加言語の対応を容易にする
- [ ] マッピングにない言語キーが来た場合でもクラッシュせず、デフォルト表示（アイコン or プレースホルダー）にフォールバックする

## スコープ外（今回やらない）

- レッスン詳細ページ（`lessons/[id]`）・カテゴリ詳細ページへの言語バナー適用
- 管理画面（`admin/*`）のUI改修（ロゴ差し替えを除く）
- 画像の手動ダークモード切り替え対応（既存のダークモード方針同様、`prefers-color-scheme`ベースで背景色側を調整するのみ）
