# Phase 2 Implementation Plan: Code Execution & Admin Expansion

## 1. Foundation: コード実行エンジンの構築 (Browser-side)
ブラウザ上でJavaScriptを安全に実行するための基盤を構築します。

- [x] 1.1 Code Runner コンポーネントの設計
  - Iframe または Web Worker を使用した隔離環境のプロトタイプ作成
  - メインスレッドとのメッセージング（実行・結果送信）の実装
- [x] 1.2 Console キャプチャの実装
  - `console.log`, `console.error` をオーバーライドしてメインUIに通知する仕組み
- [x] 1.3 タイムアウト・安全対策の導入
  - 実行時間の監視と強制終了（無限ループ対策）の実装

## 2. Core: コンソールUIとエディタ連携
実行結果を表示するUIを統合します。

- [x] 2.1 コンソールUIコンポーネントの作成
  - ログのリスト表示、エラーの赤色表示、クリアボタンの実装
- [x] 2.2 演習画面への統合
  - エディタとコンソールのレイアウト調整
  - 「実行」ボタン押下時に Code Runner を起動する連携処理

## 3. Core: 管理者向けユーザー管理の拡張 (CRUD)
バックエンドAPIとフロントエンドUIを拡張します。

- [x] 3.1 バックエンド: ユーザー編集・削除APIの追加
  - `AuthController@update`, `AuthController@destroy` の実装
  - 管理者権限チェックの適用
- [x] 3.2 フロントエンド: ユーザー編集フォームの作成
  - ユーザー名、メール、ロールを変更できるモーダルまたはページ
- [x] 3.3 フロントエンド: ユーザー削除・新規作成機能
  - 削除確認ダイアログの実装
  - 管理者によるユーザー手動作成機能

## 4. Core: サイドバーナビゲーションの導入
画面遷移を一元化し、UXを向上させます。

- [x] 4.1 Sidebar コンポーネントの作成
  - Shadcn UI (ボタン/アバター等) または Lucide アイコンを使用したデザイン
  - 認証状態および役割（Admin/User）に応じた動的リンク生成
- [x] 4.2 レイアウトへの統合
  - `layout.tsx` でサイドバーとメインコンテンツの配置を調整
  - 演習画面などの全画面コンテンツでの表示切り替えロジック
- [x] 4.3 ナビゲーションのリファクタリング
  - 各ページに散らばっているヘッダー/ナビゲーションの整理

## 5. Integration: 最終テスト
- [ ] 4.1 コード実行機能の検証
  - 正常なコード、エラーを含むコード、無限ループコードのテスト
- [ ] 4.2 ユーザー管理機能の検証
  - 権限変更、削除、重複登録のバリデーション確認
