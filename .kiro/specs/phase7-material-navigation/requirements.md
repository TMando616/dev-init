# Phase 7 要件定義: 学習資料ナビゲーション改善

## ステータス: レビュー待ち

## 概要

現状、ユーザー（生徒）が学習資料（materials）にアクセスする手段はカテゴリページ（`/categories/[id]`）経由のみであり、演習ページからは一切辿り着けない。また、レッスン一覧はフラット表示で、どのカテゴリの問題か把握しづらい。本フェーズでは「演習前に資料を読んでから問題を解く」導線を整備し、レッスン一覧をカテゴリ別に整理する。

---

## ユーザーストーリーと受け入れ条件

### US-1: 演習ページで関連資料を参照できる

**As a** 生徒
**I want** 演習ページで、その問題に関連する学習資料をその場で確認できる
**So that** 問題を解く前に必要な知識を確認でき、外部ページに離脱せずに演習に集中できる

#### 受け入れ条件

1. When 演習ページ（`/lessons/[id]`）を開いたとき、the System shall 左ペイン上部に「参考資料」セクションを表示する（関連資料が1件以上存在する場合のみ）
2. When 参考資料セクションに表示された資料カードをクリックしたとき、the System shall モーダルを開き、資料本文を Markdown レンダリングして表示する
3. When モーダルが表示されているとき、the System shall 閉じるボタンまたはモーダル背景クリックでモーダルを閉じられる
4. 関連資料は、そのレッスンに紐づくカテゴリが持つ資料を全て対象とする（多対多経由）
5. 関連資料が0件の場合、「参考資料」セクション自体を表示しない

---

### US-2: 演習ページからカテゴリへ戻れる

**As a** 生徒
**I want** 演習ページのヘッダーの戻るボタンでカテゴリページに戻れる
**So that** 同じカテゴリの他の演習問題に迷わず移動できる

#### 受け入れ条件

1. When 演習ページのヘッダーの戻るボタンをクリックしたとき、the System shall そのレッスンが属するカテゴリページ（`/categories/[id]`）へ遷移する
2. レッスンが複数カテゴリに属する場合は、`categories[0]`（最初のカテゴリ）のページへ遷移する
3. レッスンがどのカテゴリにも属さない場合は、従来通りダッシュボード（`/`）へ遷移する

---

### US-3: レッスン一覧をカテゴリ別に閲覧できる

**As a** 生徒
**I want** レッスン一覧画面でカテゴリごとにレッスンがまとまって表示される
**So that** 学習したいカテゴリの問題を素早く見つけられる

#### 受け入れ条件

1. When レッスン一覧ページ（`/lessons/list`）を開いたとき、the System shall レッスンをカテゴリ別のセクションに分けて表示する
2. 複数カテゴリに属するレッスンは、該当する全てのカテゴリセクションに重複して表示する
3. どのカテゴリにも属さないレッスンは「その他」セクションにまとめて表示する
4. カテゴリの並び順は API から返却される順序に従う
5. カテゴリが1件も取得できない場合は、現行のフラット表示にフォールバックする

---

## 技術メモ（実装者向け）

- **バックエンド変更**:
  - `LessonRepository::find()` の eager load を `categories` → `categories.materials` に拡張し、演習ページが資料情報を1リクエストで取得できるようにする
  - `GET /lessons` は現状カテゴリ付きで返却済みのため変更不要
- **フロントエンド変更**:
  - `(student)/lessons/[id]/page.tsx`: 参考資料セクション・モーダル追加、Backボタンのリンク先修正
  - `(student)/lessons/list/page.tsx`: カテゴリ別グルーピング表示（フロント側でデータを加工）
- **API追加呼び出し不要**: カテゴリ一覧は `/categories` で取得済みのものを再利用可能
